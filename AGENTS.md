## OVERVIEW

Jekyll 4.3 static site for the MSU SPARK Group (Spectroscopy, Plasmas, and Reaction Kinetics lab). Forked from [greenelab/lab-website-template](https://github.com/greenelab/lab-website-template). Ruby/Bundler build, Liquid templates, SCSS styles, custom Ruby plugins, Python citation pipeline.

## WHERE TO LOOK

| Task | Location | Notes |
|------|----------|-------|
| Site config / nav order | `_config.yaml` | Collections, plugins, front-matter defaults |
| Page layout / HTML structure | `_layouts/default.html` → `_includes/` | default wraps all pages |
| Add a new page section | `_includes/section.html` | Section breaks parsed by `content.html` via regex |
| Reusable UI component | `_includes/{name}.html` | Most accept params — see `_includes/AGENTS.md` |
| Theme colors / CSS vars | `_styles/-theme.scss` | Light/dark mode vars, font families |
| Add client JS | Drop file in `_scripts/` | Auto-included by `_includes/scripts.html` |
| Add Liquid filter | `_plugins/*.rb` | Custom filters: `regex_scan`, `regex_strip`, `array_carve`, `array_filter`, `data_filter`, `file_exists`, `file_read`, `is_nil` |
| Citation data pipeline | `_cite/cite.py` | Python, run before Jekyll build — see `_cite/AGENTS.md` |
| Add team member | `_members/{slug}.md` | Front matter: name, image, role, links, aliases |
| Add news post | `_posts/YYYY-MM-DD-{slug}.md` | Standard Jekyll posts |
| Add course | `_courses/YYYY-MM-DD-{slug}.md` | Collection with output: true |
| Publication sources | `_data/sources.yaml` | DOIs and metadata; `_data/citations.yaml` is auto-generated |
| Icon/type mapping | `_data/types.yaml` | Maps roles, social links, alert types to icons |
| CI / deploy | `.github/workflows/on-push.yaml` | Push to main → update citations → build → deploy to GH Pages |

## CONVENTIONS

- **sass_dir is `_styles`** (not Jekyll default `_sass`). All SCSS imports reference this.
- **Numbered content dirs** (`01Research/` through `07Facilities/`) — prefix controls nav order via `nav.order` in front matter.
- **Section breaks** — `{% include section.html %}` inserts HTML comments parsed by `_includes/content.html` using `regex_scan`. Sections support `dark`, `background`, `size` params.
- **Auto-include scripts** — Any `.js` in `_scripts/` is auto-loaded. No manual `<script>` tags needed.
- **No packaged theme** — `theme: null` in config. All layout HTML lives in this repo.
- **Collections** — `members`, `posts`, `courses` with `output: true`. Front-matter defaults set layouts per collection.
- **Member slugs** — Filename slug must match; used for portrait lookups and URL generation.
- **Third-party JS** — Loaded via CDN in `_includes/scripts.html`: Popper.js, Tippy.js, Mark.js.

## ANTI-PATTERNS (THIS PROJECT)

- **DO NOT edit `_data/citations.yaml` by hand** — auto-generated by `_cite/cite.py` from `_data/sources.yaml`. Edit sources instead.
- **DO NOT rename `_styles/` to `_sass/`** — config explicitly sets `sass_dir: _styles` and `include: [_styles]`.
- **DO NOT add `<script>` tags in layouts** — drop `.js` files in `_scripts/` for auto-inclusion.
- **DO NOT use spaces in filenames** — some existing members have spaces (legacy); new files should use lowercase hyphenated slugs.

## COMMANDS

```bash
# Install dependencies
bundle install

# Local development
bundle exec jekyll serve

# Production build
JEKYLL_ENV=production bundle exec jekyll build

# Update citations (Python pre-build)
pip install -r _cite/requirements.txt
python _cite/cite.py

# Full CI pipeline (what on-push does)
python _cite/cite.py && bundle exec jekyll build
```
